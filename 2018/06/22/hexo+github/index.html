hexo+github搭建个人博客

发表于 2018-06-22 | 分类于 好物推荐（技术类）

偶然在简书的首页看到了hexo搭建个人博客并托管在github的个人域名下的技术博客，据我猜想应该是将本地端口映射到了github上的相应域名，上次用花生壳的时候怎么没想到用这种方法弄自己的博客呢？不过现在好了，这个方案十分省心，昨天实践了一番弄好了自己的博客，又远程指导某杠精弄好了他的博客，在这里写下最新的配置方案，以供参考。本方案基于Macos,下文中统一用yaoyao0127来表示github用户名，实际操作时换成你自己的用户名即可。

  2. 安装git。我的git已经装了很久了，所以没有重新安装，不会的同学可以百度一下会有很多教程。

  3. 全局安装hexo。

  6. 然后安装一个往github上push的依赖包。

  7. 然后在github上新建一个仓库叫做yaoyao0127.github.io。

  8. 现在分别介绍以后常用的三条指令。hexo g表示编译或者打包项目，hexo s表示在本地端口运行该项目，hexo d表示将项目部署到github上。我们现在可以一一运行这三条指令，当本地端口预览到正常显示的博客时，就可以打包部署到github上面了。这里需要注意的一点是部署了之后不会马上生效，所以如果已经部署了以后github上yaoyao0127.github.io仓库里面的setting里面的pages项还是Your site is ready to be published ，不要着急，稍等一会儿就好了。
  9. 如果前面全部顺利的话到这里你已经拥有了一个自己的部署在github上的博客，现在我们来按照自己的喜好修改一下它的样式结构等。首先介绍一下两个配置文件，第一个就是我们前面修改过的项目根目录下的_config.yml文件，这是一个整个项目的配置文件，控制一些全局的配置。除了上面配置的url和deploy之外的一些关键的配置信息如下。

这里的next是我自己定义的主题，这里默认你也想换成这个主题，当然更换其他的主题也是类似的方法。

在项目根目录下运行这条指令，就可以从github上下载next主题的文件。下载好了以后你会发现/themes文件夹下多了一个next文件夹，next里面还有一个_config.yml文件，这就是我们要说的第二个配置文件，这个文件是配置next这个主题的一些配置项的文件，关于主题的样式、排版等设置都可以在这个文件中进行配置。参考英文注释和下面的配置项，相信聪明的你一定可以看懂并且配置好属于你自己的主题。还有很多第三方服务可以设置，不过我暂时都没有开启，因为喜欢简单的风格。

其中头部的信息很重要，categories对应的就是这篇文章的分类，在文章头部添加了分类就相当于把博客放到这一类里面了。不过到这里还需要用hexo指令给博客首页添加一个分类选项。

此时，可以看到source文件夹下面多了一个categories文件夹，就是分类这一选项对应的文件，然后还要去next文件夹下的_config.yml文件中修改一下配置项，将menu中categories的注释删掉，也就是表示在博客首页的选项中添加分类这一选项。

类似地，如果想要添加关于的选项，只要在主题的配置文件中的menu中删掉about的注释，然后用hexo new page about指令即可创建成功。
