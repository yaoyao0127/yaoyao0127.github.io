html和css的一些小问题

发表于 2018-07-20 | 分类于 html+css

###html语义化

  1. 什么是语义化：根据内容的结构，选择合适的标签，便于开发者阅读以及浏览器的爬虫和机器很好地解析。
  2. 为什么要语义化
  • 为了在没有css的情况下，页面也可以呈现出内容结构。
  • 有利于和搜索引擎建立良好的沟通，有助于爬虫抓取更多的有效信息。
  • 有利于代码维护
  1. 怎么执行
  • 少用无语义的标签
  • 规范标签的使用

###css单位

  1. px：绝对单位，页面按照精准
  2. em：相对单位，基准点为父节点的大小
  3. rem：相对单位，基准点为根节点html的大小
  4. %：百分比

###居中相关

  1. 水平居中
  • 如果width可以设为固定值，左右margin设为auto
  • 如果width不可以设为固定值，则将其父元素设置为text-align为center，将该元素display设置为inline-block
  • 如果是行内元素，直接设置其父元素text-align为center
  • 父元素position设置为relative，float设置为left，left设置为50%；子元素position设置为relative，float设置为left，left设置为-50%。
  1. 垂直居中
  • 块级元素垂直居中，通过padding-top和padding-bottom相等来实现
  • 父元素的高度为定值的单行文本：将父元素高度height和line-height设置为相同
  • 父元素的高度为定值的多行文本：将父元素display设置为table，table子元素display设置为table-cell vertical-align设置为middle，子元素display设置为inline-block
  • 任意元素垂直居中：父元素position设置为relative，子元素position设置为absolute，transform设置为translateY（50%）

###position相关

  1. position:static
    position属性的默认值，默认块级元素和行级元素按照各自的特性进行显示。
  2. position:relative
    相对定位，设置好该属性后，元素会根据margin和padding相对于自己本身的位置进行偏移，但是它原本的位置依然保留，不会被其他元素占用。
  3. position:absolute
    绝对定位，设置好该属性后，元素会脱离文档流，原本的位置不再占用，会被其他元素占用。而且无论元素是行内元素还是块级元素，都会生成一个块级框。如果该元素是行内元素，设置了绝对定位以后就可以设置width和height属性了。
    绝对定位的元素一般会相对于第一个非static属性的父元素的位置进行定位。
  4. position: fixed
    fixed设置好以后，类似于absolute，但是他不是相对于不确定的父元素进行偏移，而是相对于浏览器窗口进行偏移。通常可用于将导航固定在屏幕顶部的情况。
  5. z-index失效的几种情况
  • 父元素position为relative
  • 本身的position属性为static
  • 本身有float属性
    text-align只对子元素起作用

###display相关
关于display这个属性，用过很多次，但是还是云里雾里的，每次都是整成绝对定位来达到目标。今天来认真总结一下：

  1. display:inline
    元素变成彻底的行内元素，行内元素嘛，自然是没有height和width这些属性的，所以设置也不管用了，只能靠此时元素的内容撑开。同时，padding和margin的左右属性可以产生效果，但是上下属性却会失效。
  2. display:block
    元素变成彻底的块级元素，独自占一行，不设置width的话，会默认填满父级元素的宽度。可以自由改变width和height，margin和padding值都正常起效果。
  3. display:inline-block
    元素变成不独占一行的块级元素。虽然不独占一行，但是不设置width的话，会默认填满父级元素的宽度。可以自由改变width和height，margin和padding值都正常起效果。
  • 设置inline-block，元素不会脱离文本流
  • 对于横向排列来说，用inline-block来布局更加清晰
  • 对于需要文字环绕的时候，适合用float

###BFC(block formatting contexts)块级格式化上下文

  1. BFC的创建：浮动元素、绝对定位元素、非块级盒子的块级容器(inline-block、table-cells)、overflow属性不是visible的块级盒子，这些元素会为他们的内容创建一个BFC
  2. BFC的特点：在一个BFC中，垂直方向上，盒子是从包含块顶部开始一个挨着一个布局的，两个相邻的盒子的垂直距离是由margin属性决定的，在一个BFC中的两个相邻的块级盒子的垂直外边距会产生塌陷。
    在一个BFC中，水平方向上，每个盒子的左边缘都会接触包含块的左边缘（从右向左的格式则相反）。除非出现浮动元素和其他元素相互作用的情况（当有浮动元素时，行盒可能因浮动元素而收缩，如果有盒子形成了新的BFC，那这个盒子也可能因浮动元素而变窄）
  3. BFC的应用
  • 清除元素之间的影响：想让一个元素不被另一个元素影响，可以把其中一个元素放到BFC环境中。
  • 清除内部浮动元素对父级元素的影响

###css错题本

  1. 选择器的优先级
    内联样式 > id 选择器 > 类选择器 = 属性选择器 = 伪类选择器 > 标签选择器 = 伪元素选择器
  2. 可继承的样式
    颜色，文字，字体间距行高对齐方式，和列表的样式
  3. 外边距折叠
    上下两个盒子，上面的设置了margin-bottom，下面的设置了margin-top，此时他们之间的间距并不是两个margin的值相加，而是其中比较大的那个值。
  4. BFC块级格式化上下文就是用来格式化block块级盒子的，满足下列条件之一的盒子就会变成BFC
    float的值不为none；
    overflow的值不为visible；
    display的值为inline-block、table-cell、table-caption；
    position的值为absolute或fixed；
  5. css选择器中，逗号是分组选择器，表示并列；空格是上下文选择器，表示从属；大于号是子元素选择器，表示只选择第一个符合要求的对象；加号表示相邻兄弟选择器，表示选择紧挨着兄元素元素1的弟元素元素2，并且元素1、2有着相同的父元素，是兄弟关系；[]是属性选择器，选择具有该属性的所有元素。
