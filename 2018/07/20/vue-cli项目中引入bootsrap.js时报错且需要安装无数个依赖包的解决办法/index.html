vue-cli项目中引入bootsrap.js时报错且需要安装无数个依赖包的解决办法

发表于 2018-07-20 | 分类于 react和vue

问题描述：在vue-cli生成的项目中引入bootsrap的相关文件，bootstrap.min.css可以正常引入，但是bootstrap.js文件引入以后直接报错说缺少某个依赖包，安装完之后又报错缺少另一个依赖包。。。一直安装一直报错到我安装了差不都50个依赖包还在报错的时候我终于放弃了。。。然后在网上找答案的时候看到了这个博主的解决方法https://www.cnblogs.com/maoguy/p/8683819.html
亲测有效，在这里记录一下，顺便把引入bootstrap的整个过程记录一下。

  1. 在官网下载bootstrap的安装包，将里面的css、js、font文件放到项目目录下的assets文件夹中。

  2. 终端进入项目根目录，cnpm安装一系列依赖包。

  3. 修改build目录下的webpack.base.conf.js文件，使其支持外部的css和js。

  4. 修改src目录下的main.js，在头部引入bootstrap。

  5. 我之前在上一步中引入的是bootstrap.min.js，之后就开始不停地循环运行–报错–装依赖–运行–报错–装依赖–。。。。没有找到原因，只是找到了解决办法。

  6. 解决办法的第一步就是修改webpack.base.conf.js文件，步骤3中已经说明，照着改就行。

  7. 修改bootstrap.js

  8. 现在就可以使用bootstrap啦！当时因为没有成功导入bootstrap的js文件，导致使用modal组件时无法显示组件，看来当涉及到动态样式时，只导入css文件是不行的，昨天晚上还想了那么久为什么不行，真是蠢到家啦！
