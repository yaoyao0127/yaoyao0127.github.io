前端项目中遇到的问题记录

发表于 2018-07-18 | 分类于 问题集锦
  1. 问题：webpack打包时提示找不到该模块
    解决方法：将webpack安装在项目目录下
  2. 问题：webpack打包报错
    根据错误我们可以看出来_webpack@2.3.2@webpack，版本是2.3.2，错误信息说的是“configuration.resolve.extensions[0] should not be empty”。这里就看出来说的是configuration.resolve.extensions[0]不需要配置空的。根据错误信息，我们就知道了，这个是2.x版后就不需要配置为空的那个了
  3. package.json中的name的值不可以和npm安装的依赖包的名字一样
  4. webpack配置文件中，引入css时，写成css会报错，必须写成css-loader
  5. sublime装了emment插件以后，需要把文件类型换成html后才能使用快捷tab方式
  6. sublime编辑器中，JSX语法无法高亮显示的解决方法：
    下载babel插件，打开.jsx文件，选择view-Syntax-Babel-JavaScript（babel）即可。
  7. app,js文件中添加的注释语句被显示到了页面中的解决方法：
    因为<layout><div>这类标签内的内容会显示出来，因此通常的 //这是一条注释 以及 /*这是一条注释*/ 这两种方法中的注释语句都会被解析出来，采用{/*这是一条注释*/}即可。
